# Wiki: Hướng dẫn Triển khai Kubernetes (K8s) trên Rocky Linux 9 (VMware)

Tài liệu này tổng hợp quy trình dựng cụm K8s với kiến trúc **1 Master - 2 Worker** trên môi trường ảo hóa VMware Workstation.

---

## 1. Chuẩn bị Hạ tầng (VMware)
- **Hệ điều hành**: Rocky Linux 9 (Minimal ISO).
- **Cấu hình Master**: 2 CPU, 4GB RAM, 20GB Disk.
- **Cấu hình Worker**: 2 CPU, 2GB RAM, 20GB Disk.
- **Network**: Chế độ **NAT** (để vừa có internet vừa thông mạng nội bộ).

---

## 2. Bước 1: Chuẩn bị máy ảo Gốc (Base VM)
Để tiết kiệm thời gian, chúng ta cài đặt mọi thành phần chung lên một máy trước khi nhân bản.

### Script "All-in-one" chuẩn bị máy Gốc:
Đăng nhập root và chạy duy nhất khối lệnh này:
```bash
# 1. Tắt Swap, SELinux và Firewall
sudo swapoff -a
sudo sed -i '/ swap / s/^\(.*\)$/#\1/g' /etc/fstab
sudo setenforce 0
sudo sed -i 's/^SELINUX=enforcing$/SELINUX=permissive/' /etc/selinux/config
sudo systemctl disable --now firewalld

# 2. Cấu hình Mạng hệ thống
cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF
sudo modprobe overlay
sudo modprobe br_netfilter
cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF
sudo sysctl --system

# 3. Cài đặt Containerd (Container Runtime)
sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo dnf install -y containerd.io
sudo mkdir -p /etc/containerd
containerd config default | sudo tee /etc/containerd/config.toml >/dev/null
sudo sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml
sudo systemctl enable --now containerd

# 4. Cài đặt Kubeadm, Kubelet, Kubectl (v1.30)
cat <<EOF | sudo tee /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=https://pkgs.k8s.io/core:/stable:/v1.30/rpm/
enabled=1
gpgcheck=1
gpgkey=https://pkgs.k8s.io/core:/stable:/v1.30/rpm/repodata/repomd.xml.key
EOF
sudo dnf install -y kubelet kubeadm kubectl --disableexcludes=kubernetes
sudo systemctl enable --now kubelet

# 5. Cấu hình cho phép Root SSH (Để copy-paste từ Windows)
echo "PermitRootLogin yes" >> /etc/ssh/sshd_config
systemctl restart sshd
```

---

## 3. Bước 2: Nhân bản và Cấu hình định danh
1. **Tắt máy Gốc** -> Chọn **Clone** -> **Full Clone** ra 3 máy: `k8s-master`, `k8s-worker-1`, `k8s-worker-2`.
2. **Đổi Hostname** trên từng máy:
   - Master: `hostnamectl set-hostname k8s-master`
   - Worker 1: `hostnamectl set-hostname k8s-worker-1`
   - Worker 2: `hostnamectl set-hostname k8s-worker-2`
3. **Sửa file hosts**: (Để tránh cảnh báo Hostname not reached)
   - Chạy lệnh: `echo "127.0.0.1 $(hostname)" >> /etc/hosts` trên mỗi máy.

---

## 4. Bước 3: Khởi tạo Cluster (Chỉ làm trên Master)
1. **Khởi tạo**:
   ```bash
   sudo kubeadm init --pod-network-cidr=10.244.0.0/16
   ```
2. **Cấu hình Kubectl**:
   ```bash
   mkdir -p $HOME/.kube
   sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
   sudo chown $(id -u):$(id -g) $HOME/.kube/config
   ```
3. **Cài đặt mạng Pod (Flannel)**:
   ```bash
   kubectl apply -f https://github.com/flannel-io/flannel/releases/latest/download/kube-flannel.yml
   ```

---

## 5. Bước 4: Kết nối máy con (Chỉ làm trên Worker)
Dán lệnh `kubeadm join` lấy từ kết quả bước 3 vào 2 máy Worker.

---

## 6. Tổng hợp các lỗi thực tế và cách xử lý

### Lỗi 1: Không SSH được vào user Root
- **Hiện tượng**: Báo "Authentication failed" dù mật khẩu đúng.
- **Nguyên nhân**: Rocky Linux chặn Root đăng nhập SSH mặc định.
- **Xử lý**: Sửa `/etc/ssh/sshd_config` đổi `PermitRootLogin` thành `yes` và restart `sshd`.

### Lỗi 2: `kubeadm: command not found` sau khi Clone
- **Hiện tượng**: Gõ lệnh khởi tạo báo không tìm thấy lệnh.
- **Nguyên nhân**: Tắt máy và Clone quá sớm khi chưa chạy xong bước cài đặt Kubeadm trên máy Gốc.
- **Xử lý**: Phải đảm bảo máy Gốc đã cài đủ Containerd và Kubeadm trước khi thực hiện Clone.

### Lỗi 3: Node ở trạng thái `NotReady`
- **Hiện tượng**: `kubectl get nodes` thấy node nhưng status không phải Ready.
- **Nguyên nhân**: Chưa cài Network Plugin (CNI).
- **Xử lý**: Cài Flannel hoặc Calico. Đợi khoảng 1 phút để các Image mạng được tải xuống các máy con.

---

## 7. Lệnh kiểm tra nhanh
- Kiểm tra node: `kubectl get nodes`
- Kiểm tra tất cả các dịch vụ: `kubectl get pods -A`
- Xem log của một pod: `kubectl logs <pod-name> -n <namespace>`

## 8. Giải thích chi tiết các cấu hình hệ thống

Việc hiểu rõ lý do tại sao phải thực hiện các bước chuẩn bị hệ thống sẽ giúp bạn quản lý cụm K8s tốt hơn.

### 8.1. Tại sao phải tắt SWAP?
*   **Hiệu suất**: K8s được thiết kế để chạy ứng dụng trên RAM. Nếu dùng Swap (ổ cứng), tốc độ sẽ giảm đi hàng trăm lần, làm sai lệch các báo cáo về hiệu suất.
*   **Độ chính xác**: Bộ lập lịch (Scheduler) của K8s không thể quản lý chính xác tài nguyên nếu Swap còn bật. K8s muốn hệ thống "fail fast" (lỗi ngay khi hết RAM) để nó có thể di chuyển Pod sang máy khác, thay vì để ứng dụng chạy dật dờ trên Swap.

### 8.2. Tại sao phải tắt SELinux?
*   **Quyền truy cập**: SELinux kiểm soát rất chặt chẽ việc tiến trình nào được chạm vào file nào. K8s cần can thiệp sâu vào hệ thống tệp của host để quản lý Container, Log và Volume. 
*   **Đơn giản hóa**: Việc cấu hình luật SELinux cho K8s cực kỳ phức tạp. Chuyển sang `permissive` giúp K8s hoạt động trơn tru mà không bị các lỗi "Permission Denied" chặn đứng.

### 8.3. Tại sao phải tắt Firewall (Firewalld)?
*   **Xung đột mạng**: K8s sử dụng IPTables để định tuyến lưu lượng giữa các Pod. `firewalld` thường ghi đè hoặc xung đột với các quy tắc này.
*   **Cổng kết nối**: K8s cần mở rất nhiều cổng (6443, 2379, 10250...). Việc tắt Firewall giúp đảm bảo các máy Master và Worker luôn thông suốt 100%. Trong môi trường chuyên nghiệp, người ta sẽ cấu hình Security Group ở lớp mạng bên ngoài thay vì dùng firewall trên chính server.
